Для запуска и сборки контейнера на вашем VPS (если у вас Debian, Ubuntu или аналогичный дистрибутив) нужно установить всего три вещи: **Git** (для скачивания файлов), **Docker** (основа) и плагин **Docker Compose**.

Выполните эти команды по очереди от имени пользователя `root`:

### Шаг 1: Обновление системы и установка Git
Сначала обновим списки пакетов и установим Git и Curl:
```bash
apt update && apt upgrade -y
apt install -y git curl ca-certificates
```

### Шаг 2: Установка Docker (официальный способ)
Самый надежный и быстрый способ установить актуальную версию Docker на любой Linux — использовать официальный скрипт от разработчиков
```bash
curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh
```

### Шаг 3: Установка плагин-клиента Docker Compose (V2)
Установим официальный плагин Docker Compose, чтобы вы могли использовать команду `docker compose`:
```bash
apt install -y docker-compose-plugin
```

### Шаг 4: Запуск и проверка автозагрузки
Включим автозагрузку Docker, чтобы контейнеры поднимались при перезагрузке самого сервера VPS:
```bash
systemctl enable --now docker
```

---

### Шаг 5: Проверка успешности установки
Вы можете убедиться, что всё готово к работе, выполнив команды:
```bash
git --version
docker --version
docker compose version
```

Чтобы скачать только нужную вам папку `docker-wdtt` без загрузки всего остального репозитория, выполните по очереди эти команды:

```bash
# 1. Скачиваем только структуру последнего коммита, не извлекая файлы
git clone --depth 1 --no-checkout https://github.com/michaillepichow/proxy-turn-vk-android-server-fix.git

# 2. Переходим в созданную папку репозитория
cd proxy-turn-vk-android-server-fix

# 3. Указываем гит, что нам нужна ТОЛЬКО папка docker-wdtt
git sparse-checkout set docker-wdtt

# 4. Извлекаем файлы на диск
git checkout

# 5. Переходим в готовую рабочую папку с файлами докера
cd docker-wdtt
```





### Сценарий 1: Чистая установка (Обычный режим с нуля)

Этот вариант используется, если на сервере раньше не было этого VPN, либо вы ставите его на новый «чистый» VPS.

#### 1. Что должно лежать в папке на сервере:
*   **`Dockerfile`**
*   **`entrypoint.sh`** (с правами на запуск: `chmod +x entrypoint.sh`)
*   **`docker-compose.yml`**
*   **`.env`** (этот файл вы создаете сами)

#### 2. Что нужно сделать для запуска:
1. Создайте и заполните файл **`.env`**, указав ваши порты и настройки (токен бота, пароли) в переменной `WDTT_ARGS` :
   ```env
   WDTT_DTLS_PORT=56000
   WDTT_WG_PORT=56001
   WDTT_CLIENT_CIDR=10.66.66.0/24
   WDTT_ENABLE_TCPMSS=1
   WDTT_ARGS="-password 'ВашПароль' -bot-token '123:ABC' -admin-id '98765'"
   ```
2. Запустите сборку и старт одной командой:
   ```bash
   docker compose up -d --build
   ```
*Все данные (пароли пользователей) сервер автоматически создаст и сохранит в папке `./config`.*

### Сценарий 2: Миграция (Переезд с работающего systemctl)

Этот вариант используется, только если у вас прямо сейчас на сервере крутится старый VPN в виде службы `systemd`, и вы хотите перенести его в Docker без потери пользователей.

#### 1. Что должно лежать в папке на сервере:
*   **`Dockerfile`**
*   **`entrypoint.sh`** (с правами на запуск: `chmod +x entrypoint.sh`)
*   **`docker-compose.yml`**
*   **`migrate.sh`** (с правами на запуск: `chmod +x migrate.sh`)
*   *(Файл `.env` заранее создавать **не нужно** — скрипт миграции создаст его сам, скопировав настройки из старой службы).*

#### 2. Что нужно сделать для запуска:
1. Запустите скрипт миграции от имени `root`:
   ```bash
   sudo ./migrate.sh
   ```
2. Скрипт сам считает старые порты и токены, перенесет базу пользователей, выключит и удалит старую службу `systemctl` и запустит Docker.
3. **Важный шаг в конце:** Из-за особенностей очистки сетевых интерфейсов, сразу после окончания работы скрипта один раз перезапустите контейнер, чтобы восстановить сеть:
   ```bash
   docker compose restart
   ```






Внутри образа — только **Debian** и **3 ключевых компонента**:

1. **`wdtt-server`** (в `/usr/local/bin/`) — сам бинарник VPN с GitHub.
2. **Скрипт запуска** (`/entrypoint.sh`) — настраивает NAT и включает сервер.
3. **Три сетевые утилиты**:
   * `iptables` — раздает интернет клиентам (NAT/MASQUERADE).
   * `iproute2` — создает сетевой интерфейс `wdtt0`.
   * `ca-certificates` — обеспечивает безопасное соединение для Telegram-бота.

Все пароли и ключи хранятся снаружи в папке `./config` на хосте и пробрасываются в контейнер по пути `/etc/wdtt` .




Вот краткая шпаргалка самых полезных команд для управления и проверки сервера. Все команды выполняются из папки с проектом (где лежит `docker-compose.yml`).

### 1. Логи и активность
*   **Смотреть логи в реальном времени (все подключения и ошибки):**
    ```bash
    docker compose logs -f
    ```
*   **Показать последние 50 строк лога и следить за новыми:**
    ```bash
    docker compose logs -f --tail 50
    ```

*   **Посмотреть статус контейнера (работает ли сейчас):**
    ```bash
    docker compose ps
    ```

### 2. Управление сервером
*   **Перезапустить (нужно после ручного изменения `passwords.json`):**
    ```bash
    docker compose restart
    ```
*   **Остановить и выключить сервер:**
    ```bash
    docker compose down
    ```
*   **Включить обратно в фоновом режиме:**
    ```bash
    docker compose up -d
    ```
